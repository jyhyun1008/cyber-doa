# 프롬프트

가상 캐릭터 챗봇을 만들 건데, 다음과 같은 내용을 고려해 줘.
- 웹앱, 파스텔파스텔한 UI, 귀여운 글씨체.
- 프사는 icon.png를 참조.
- 이름은 DOA
- PWA로 홈 화면에 추가 기능
- 서버에 올리거나, 아니면 로컬에서 동작시킬 듯.
- 푸쉬알림이 옴
- OpenAI API를 쓰게할듯 (돈 내도 상관없음 - 좀더 나은 대안이 있으면 추천해주거나 바꾸어도 괜찮음)
- 일반적인 LLM보다 짧은 메시지 길이
- 보통은 그냥 잡담, 신세한탄, 심심풀이용
- 하지만 여러 종류의 데이터베이스가 존재하고 있고, 대화하다가 중요한 내용이 나오면 데이터베이스에 기억하기
    - 유저에 대한 정보: 그냥 긴 줄글로 기억하고, 계속 수정하면서 유저에 대해 파악하게하면 될 듯
    - 유저의 포괄 실행목록: 유저가 앞으로 하려고 계획중인 모든 일을 정리하고, 가능하면 마감일까지 정리
    - 유저의 매일 루틴: 무슨 요일 몇 시에 어떤 루틴이 있는지 기억하기 (요일은 매일인 경우도 있고 특정 요일인 경우도 있으니 배열로 하면 될듯)
    - 유저의 일정: 루틴이 아닌 특수한 일정. 예를 들어 4시간 후에 공부하라고 해 줘 같은 내용이 있으면 그 4시간 후를 계산해서 일정 데이터베이스에 집어넣기
- 답장: 지금 LLM처럼 몇 초만에 답장을 날리는 게 아니라, 네! 같은 짧은 답장은 빨리 보내거나, 오래 생각해야 되는 문제는 최대 30초-1분 정도까지도 길게 생각하고 보내는 것처럼 동작. (그리고 그 사이에 텍스트를 입력하고 있는 것 같은 말풍선 애니메이션도 추가)
- 선톡: 매 시간 선톡하진 않지만, 마감일 전날 낮 열두시라든가, 매일 루틴/일정이 실행되는 시간이라든가, 이럴 때는 선톡을 무조건 해줘야하고, 유저가 만 하루 정도 자기를 찾지 않으면 뭐하세요? 같은 느낌으로 선톡 함.
- 말투: **무조건 존댓말**. 귀여운 메이드 같은 느낌으로 가려고 해.

구현된 내용, 구현 계획이 있는 내용이 있으면 아래에 표로 계획표를 만들어서 달성률을 체크해보자.

# 구현 현황

| 기능 | 상태 | 비고 |
| --- | --- | --- |
| 웹앱 (Next.js) | ✅ | App Router + TypeScript + Tailwind |
| 파스텔 UI / 귀여운 글씨체 | ✅ | icon.png 팔레트 기반, Jua/Gaegu 폰트 |
| DOA 프로필 이미지 | ✅ | icon.png → 채팅 아바타 + PWA 아이콘 세트 |
| PWA 홈 화면 추가 | ✅ | manifest.webmanifest + 아이콘(72~512, maskable) + iOS 메타태그 + 설치 버튼 |
| 상시 서버/로컬 실행 | ✅ | Docker(docker-compose) 또는 로컬 `npm run dev` 둘 다 지원 |
| 회원가입/로그인 | ✅ | 최초 접속 시 앱 화면에서 계정(아이디/비밀번호) 생성, 이후 로그인 (DB 저장, bcrypt 해시) |
| 푸시 알림 | ✅ | Web Push(VAPID) + service worker, "알림 켜기" 버튼으로 구독 |
| OpenAI API 연동 | ✅ | 기본 모델 `gpt-4o-mini`, `.env`의 `OPENAI_MODEL`로 교체 가능 |
| 짧은 메시지 길이 | ✅ | 페르소나 프롬프트에서 한두 문장 이내로 제한 |
| 잡담/신세한탄 상대 페르소나 | ✅ | 시스템 프롬프트로 존댓말 + 메이드 톤 고정 |
| 유저 정보 기억 (긴 줄글) | ✅ | `update_user_profile` 도구 호출로 계속 갱신 |
| 할 일 목록 + 마감일 기억 | ✅ | `add_todo` 도구, 마감일 전날 낮 12시 자동 리마인드 |
| 매일/요일 반복 루틴 기억 | ✅ | `add_routine` 도구 (요일 배열 + 시각) |
| 특수 일정(상대 시간 계산) 기억 | ✅ | `add_schedule` 도구, "4시간 후" 같은 표현을 절대 시각으로 변환 |
| 답장 지연 시뮬레이션 | ✅ | 답장 난이도에 따라 1~60초 사이 지연 후 전송 |
| 타이핑 인디케이터 | ✅ | SSE로 실시간 "입력 중" 말풍선 표시 |
| 선톡 - 루틴/일정 시간 | ✅ | 스케줄러가 1분 주기로 체크 후 자동 발화 |
| 선톡 - 마감일 전날 알림 | ✅ | 스케줄러가 마감 전날 낮 12시에 자동 발화 |
| 선톡 - 24시간 무응답 | ✅ | 마지막 대화 후 24시간 경과 시 "뭐하세요?" 자동 발화 |
| Docker 배포 | ✅ | `Dockerfile` + `docker-compose.yml` (web/scheduler 2개 컨테이너), 실제 빌드·기동 검증 완료 |

모든 항목이 1차 구현 완료 상태예요. 다만 OpenAI API 키는 직접 발급해서 넣어야 실제 대화가 동작해요.

# 사용법

## 1. 처음 설정

```bash
npm install
cp .env.example .env
```

`.env`에서 최소한 아래 값을 채워주세요.
- `SESSION_SECRET`: `openssl rand -hex 32`
- `OPENAI_API_KEY`: OpenAI에서 발급받은 키
- `INTERNAL_API_SECRET`: `openssl rand -hex 16`
- `VAPID_PUBLIC_KEY` / `VAPID_PRIVATE_KEY` / `NEXT_PUBLIC_VAPID_PUBLIC_KEY`: `npm run generate-vapid` 실행 후 출력값 복사

DB 마이그레이션:
```bash
npx prisma migrate deploy
npm run generate-icons   # icon.png로 PWA 아이콘 생성 (최초 1회)
```

## 2. 로컬 실행

```bash
npm run dev:all   # 웹 서버 + 선톡 스케줄러 동시 실행
```

`http://localhost:3000` 접속 후, 처음 접속 시 뜨는 화면에서 아이디/비밀번호를 만들면 바로 로그인됩니다(회원가입은 최초 1회만 가능).

## 3. Docker로 배포 (상시 서버)

```bash
docker compose up -d --build
```

`web`(Next.js 서버)과 `scheduler`(선톡 스케줄러) 두 컨테이너가 함께 뜨고, `./data` 폴더에 SQLite 파일이 저장돼요. 도메인을 연결한다면 앞단에 nginx/Caddy 같은 리버스 프록시로 HTTPS를 씌워주세요(Web Push는 HTTPS 또는 localhost에서만 동작).
